Commit message format

Each commit message has this format :

<type>(<scope>): <subject>
<BLANK LINE>
<body>
<BLANK LINE>
<footer>

The header (first line) is mandatory with a maximum of 50 characters.

Body and footer are optional and each line cannot be longer than 70 characters.

Type is one of: * feat: add/remove/update a feature * fix: fix a bug * docs: add/remove/update documentation * test: add/remove/update test code * refactor: regorganisation of files and folders * style: formatting code

Scope list: * common * common/servers * common/services * common/rendering * common/resources * common/runtime * common/security * server * browser * plugins
